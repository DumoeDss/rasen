**Automatic session relay** — the "platform cannot restart the main session" assumption is retired (verified 2026-07-07, claude CLI 2.1.202): an exhausted LEAD session can now launch its own successor. The `handoff` workflow gains a Session relay protocol: after the handoff document and run-state hit disk, and with the user's authorization, it spawns a visible interactive Claude Code window bootstrapped to read the distillate, run `openspec pipeline resume`, and continue from the documented next action. Bootstrap prompts travel via file indirection (`handoff/relay-prompt.txt`) or PowerShell `-EncodedCommand` — bare-quoted prompts are truncated by nested shell parsing (observed live). Invariants in the orchestration playbook (new Step H.7): relay only at a stage boundary (quiesce — every worker returned, run-state persisted), spawn-then-stand-down ordering, a relay generation cap (`sessionHandoff.n` at `maxRelays` stops auto-relay and recommends decomposing), and no cross-session worker resurrection (successors rebuild via the Step F.1 ladder). The `/opsx:auto` pre-flight upgrades from a one-line reminder to a three-way offer (automatic relay / continue / manual). Run-state `sessionHandoff` accepts an optional generation `n` (absent = 1) and `pipeline resume` reports it in both text and JSON. New optional compact-recovery hook (`hooks/compact-recovery.sh`, SessionStart matcher `compact`; `openspec init` prints the snippet, never edits `.claude/settings.json`) re-anchors a freshly compacted session on the handoff distillates instead of the machine summary.
