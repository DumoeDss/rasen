**Context sensing & handoff** — agents can now measure context-window occupancy and hand work to a successor instead of silently degrading. New `openspec agent context [--latest|--transcript <path>] [--json]` reads exact usage from transcript-recorded API data (no estimation). New opt-in `handoff` workflow (`/opsx:handoff`, skill `openspec-handoff`) writes a distilled handoff document (decisions, dead ends, eliminated hypotheses, next action) and records a `sessionHandoff` pointer in run-state. The orchestration playbook gains Step H: worker dispatch prompts carry a self-handoff contract (compaction marker / soft budget triggers, structured `HANDOFF` return, LEAD-side accounting and successor spawning in the same session), a warm-continue guard probes worker transcripts before every `SendMessage` (retire-via-handoff above threshold — makes the planner's "retire on bloat" deterministic), relay caps (`maxRelays`, default 3) with stall detection (`stallLimit`, default 2; eliminating a hypothesis counts as progress) trigger a LEAD strategy review — not a human gate — and exhausted budgets park the stage as `escalated` (non-blocking; surfaced at the next gate/run end). Review-loop round exhaustion routes through the same ladder instead of an immediate human stop; "never silently pass" is unchanged. Pipeline YAML accepts a `handoff` config block (pipeline + stage level, per-role thresholds; resolution stage > roles[role] > pipeline > defaults `{threshold: 0.5, maxRelays: 3, stallLimit: 2}`), `pipeline show` reports resolved values, run-state accepts optional `sessionHandoff`/per-stage `handoffs[]`, and `pipeline resume` surfaces handoff pointers plus per-worker context estimates. `/opsx:auto` performs a one-shot non-blocking pre-flight probe of the LEAD session.
